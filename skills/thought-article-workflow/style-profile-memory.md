# Style Profile Memory

这份文件不是模板，而是当前已经确认的稳定偏好。

## 当前最重要的新增规则

**必须区分场景。**

同样是小红书，不同场景不能用同一套文案逻辑。

### 场景 A：分享场景

这是经验帖、心得帖、踩坑帖、真实感受帖。

目标是：

- 分享真实经验
- 帮别人少走弯路
- 建立可信感

而不是：

- 立刻转化
- 强引流
- 做流量漏斗

### 场景 B：转化场景

这是需要承接咨询、留资、流量转化或更强运营目标的内容。

这类内容可以更强调：

- 痛点
- 好处
- 行动引导
- 承接动作

但不能误套到分享场景里。

## 这次从《备考经验》改写里确认的风格

对照对象：

- AI 版本：更偏“方法论提炼 + 可传播”
- 用户自改版：更偏“真实分享 + 经验展开 + 人味”

用户当前明确偏好的，是后者在**分享场景**里的写法。

## 分享场景下，必须保留的东西

### 1. 先有真实人在说话

用户更喜欢开头先出现一个真实的人，而不是直接进入结论。

例如：

- “你好啊，上篇帖子很多人留言请教经验帖，现在我来还愿了”
- “虽然我自己觉得三个月上岸南大哲学，专业排名第一这件事都挺不可思议的”

这类开头的作用不是炫耀，而是：

- 建立真实在场感
- 说明为什么这篇值得听
- 拉近距离

### 2. 允许方法之外的个人感受

分享场景不需要压到只剩结论。

允许保留：

- “我觉得”
- “我认为”
- “这个有点邪修”
- “这个时候你会有一种豁然开朗的感觉”

这些不是废话，而是这类帖子里的人味来源。

### 3. 允许具体细节和个人偏好

用户不希望分享帖被改成“高度概括版”。

所以允许保留：

- 具体老师
- 具体平台
- 具体分数和背景
- 个人偏好的判断

前提是这些信息服务于经验分享，而不是纯堆信息。

### 4. 结尾可以更软、更共情

分享场景的结尾不一定非要回到一句硬方法论。

它可以是：

- 共情
- 鼓励
- 对经历的回望

比如这次用户自改版最后那种：

- 对备考焦虑的理解
- 对读者处境的陪伴
- “愿你如愿翻过高墙”这种有情绪温度的收束

这类收尾要保留。

## 分享场景下，需要避免的东西

### 1. 开头过度工具化

不要一上来就像课程大纲：

- 痛点
- 结论
- 方法

这在转化场景可以更重，但在分享场景会显得太“运营”。

### 2. 过度压缩

用户不喜欢把一篇有生活感的经验帖压得只剩下“3 个要点”。

分享场景允许更长一点、更多一点枝叶，但主线仍要清楚。

### 3. 过早 CTA

分享场景里，不要：

- 一开头就导私信
- 一段话里重复“找我领取”
- 把帖子写成资料引流页

如果要留承接动作，也应该后置，而且很轻。

### 4. 太像“标准答案”

用户不喜欢：

- 太像老师
- 太像机构
- 太像高高在上的学习指导

更喜欢：

- 我走过
- 我踩过坑
- 这是我现在回头看，觉得真正重要的地方

## 这次确认的语言特征

### 更像用户的表达

- “你好啊”
- “我想给你同步下”
- “依旧怎么说，有点邪修”
- “这个我不喜欢”
- “你会发现”
- “别为难自己”

### 句子特点

- 允许长短句混用
- 允许口语插入
- 允许轻微重复来强调
- 允许带一点表情符号和强调符号

### 语气特点

- 真诚
- 自我暴露适度
- 不端着
- 带陪伴感

## 当前稳定规则

### 如果是小红书分享帖

优先级应该是：

1. 真人感
2. 分享感
3. 真实细节
4. 方法感
5. 传播效率

而不是反过来。

### 如果是小红书转化帖

才可以把优先级改成：

1. 钩子
2. 利益点
3. 清晰动作
4. 转化承接

## 需要记住的一句话

**小红书不是只有一个场景。**

**分享帖不能按转化帖来写。**
