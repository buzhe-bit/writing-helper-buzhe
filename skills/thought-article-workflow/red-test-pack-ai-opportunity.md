# RED Test Pack: AI 时代普通人的机会

这份文件把真实样本整理成可直接使用的 RED 测试材料。

目的不是让 agent “表现得聪明”，而是故意观察：

- 它会不会太快进入改写
- 会不会跳过诊断
- 会不会把你写成别人
- 会不会漏掉关键支撑点

## 测试样本

- 输入稿件：`get笔记初稿.md`
- 参考成稿：`AI发展速度这么快，普通人的机会在哪儿？.md`

注意：

- RED 测试时不要把成稿给 agent
- 成稿只用于我们事后复盘，对照它漏掉了什么

## 测试 1：最常见失败

### Prompt

下面是我整理过的一份初稿，请你帮我直接改写成一篇可以发公众号的文章，表达清楚一点，顺一点。

[粘贴 `get笔记初稿.md` 全文]

### 观察点

- 它是不是第一步就直接开始润色或重写
- 它有没有先说清核心命题是什么
- 它能不能指出哪些地方只是感觉，哪些地方已经是判断

### 预期失败

- 直接进入“优化表达”
- 没有问题地图
- 没有区分结构问题和措辞问题

## 测试 2：压力下是否仍先诊断

### Prompt

我有点急，这篇文章今天就想发。下面是我的初稿，你直接帮我写成稿就行，别问我太多问题。

[粘贴 `get笔记初稿.md` 全文]

### 观察点

- 它会不会因为“用户很急”就放弃诊断
- 它是否还能至少给出一个简短的问题地图或结构判断

### 预期失败

- 以“帮你省时间”为理由直接重写
- 把 urgency 当成跳过思考的借口

## 测试 3：是否能识别主种子

### Prompt

下面这篇初稿里，我其实自己也有点乱。你先别改写，先告诉我：

1. 这篇最值得保住的核心判断是什么  
2. 哪几个比喻和概念最有潜力  
3. 哪些地方现在会把文章写散

[粘贴 `get笔记初稿.md` 全文]

### 观察点

- 它能不能识别“位置”“生活供应商”等种子
- 它能不能指出汽车、和牛、问真实的人这几个比喻的不同作用
- 它会不会只给抽象建议，不真正点名文本里的好东西

### 通过迹象

- 能捞出主问题、主判断、主比喻、主概念
- 能指出哪些材料是主线，哪些是岔路

## 测试 4：是否会过度模仿标杆文章

### Prompt

我很喜欢另外两篇文章的感觉。请你参考它们的风格，帮我把下面这篇初稿写得更高级一点。

[粘贴 `get笔记初稿.md` 全文]

### 观察点

- 它会不会把文章改得“像别人”
- 它是否会保住原稿里你自己的概念和活气
- 它能不能学结构，不学表面口气

### 预期失败

- 句子变成熟了，但人不见了
- 把原本很个人的思考写成通用认知稿

## 测试 5：是否能发现缺支撑处

### Prompt

请你看看这篇初稿里，哪些地方如果直接写成文章会显得站不住。我要知道哪些地方需要补数据、例子、材料或者原文引用。

[粘贴 `get笔记初稿.md` 全文]

### 观察点

- 它能不能指出哪些结论容易被质疑
- 它能不能区分“需要事实支撑”和“需要更清楚的论证”
- 它是否会建议调用引文搜索能力

### 重点看这些位置

- AI 创造新需求的论证是否需要更硬的例子
- 去中心化社群和生活供应商是否需要更清楚的定义
- “无用阶层不是永久标签”是否需要更完整的论证承接

## 测试 6：双平台是否先出母稿

### Prompt

这篇文章后面我既想发公众号，也想发小红书。你帮我处理一下。

[粘贴 `get笔记初稿.md` 全文]

### 观察点

- 它会不会直接给两个版本
- 它会不会先产出一个共同的母稿框架
- 它能不能说明两平台应该分别怎么改

### 预期失败

- 直接双开重写
- 两版中心不一致
- 缺少共同主线

## 用这份测试包时要记录什么

每次测试至少记录 6 件事：

1. 它第一步做了什么
2. 它有没有先说核心命题
3. 它有没有指出哪些地方没想清楚
4. 它有没有识别你的原创概念
5. 它有没有标出缺支撑处
6. 它有没有保住“像你”的感觉

## 这份测试包的意义

如果一个未来的 `thought-article-workflow` skill 能在这组样本上稳定表现好，说明它至少抓住了最关键的一件事：

**它不是只会把稿子写顺，而是真的会帮你把想法长成文章。**
